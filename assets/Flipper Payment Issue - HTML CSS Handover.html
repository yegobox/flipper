01 — Overview

What this screen does

A recoverable error state shown when a subscription charge fails. Rather than a dead-end, it lets the user fix the cause in place: switch or upgrade the plan, apply a discount, retry with a different MTN number, or defer. The whole screen scrolls as one column inside a phone frame with a fixed header.

  • Primary goal — recover the payment (Try Again).
  • Secondary — adjust the plan / cycle / add-on so the charge succeeds.
  • Escape hatches — Skip for Now, and Need Help (chat with support).
Files. Styles live in payment_issue/payment.css (depends on the base tokens in onboarding/styles.css). Markup + behaviour reference is payment_issue/payment-app.jsx — but this handover gives you the same UI as framework-free HTML/CSS.

18 — Data & logic

Model + total
const PLANS = [
  { id:'mobile',  name:'Mobile only',      m:5000,   y:48000   },
  { id:'desktop', name:'Mobile + Desktop', m:12000,  y:115200  },
  { id:'ent',     name:'Enterprise',       m:150000, y:1440000 },
];
const ADDON = { name:'Tax Reporting Consulting', m:30000, y:288000 };

function newTotal(plan, cycle, addon) {
  const base  = cycle === 'yearly' ? plan.y : plan.m;
  const extra = addon ? (cycle === 'yearly' ? ADDON.y : ADDON.m) : 0;
  return base + extra;
}
// grouped thousands for RWF
const fmt = n => n.toLocaleString('en-US');          // 1,440,000
// display: `RWF ${fmt(newTotal(...))}.00`  → Geist Mono, --blue, 18/800

Yearly per-month figure shown on tiles = Math.round(y / 12).

19 — Scroll & layout gotcha

Read this before you build

The screen is a flex column: a fixed .pi-top, then a .pi-scroll region with flex:1; min-height:0; overflow-y:auto. Blocks inside are separated by gap:16px.

The bug that bit this build. .pi-scroll is itself a flex column, so its children are flex items and can shrink below their content height. The plan-switcher card has overflow:hidden — when it got shrunk, it clipped its own contents to a 2px sliver and vanished. The other cards only stayed visible because they don't clip. Fix: .pi-scroll > * { flex-shrink: 0; } — keep every block at its natural height and let the container scroll. Apply this to any scroll-in-flex container, not just this screen.
