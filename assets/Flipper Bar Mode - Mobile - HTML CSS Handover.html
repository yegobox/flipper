Bar Mode — Mobile

The complete spec to build the phone / small-device version of Flipper Bar Mode — the roaming-waiter surface. Same feature and same data as the tablet register, re-laid-out for one hand on a 402 × 874 canvas: catalog and tab split apart, the running tab as a pull-up sheet, and touch targets ≥ 44px. Every value here is pulled directly from the approved prototype Flipper Bar Mode - Mobile.html.

Canvas 402 × 874 (iPhone) Currency RWF · VAT 18% Font Geist / Geist Mono Shares store with the tablet build New bar/mobile.css + bar/mobile.jsx
Read alongside the tablet handover. The data model, store actions, persistence, cashiers/products/floor, and settle maths are identical and documented in full there. This doc covers what is mobile-specific: layout, components, gestures, and the screens the phone shows.

01 — Overview

The waiter's phone

Mobile is the waiter-facing half of Bar Mode. A server carries a phone around the floor: they log in with a PIN, pick a table, tap drinks into its running tab, and pull up the tab to review, save, or hand off. Settlement is still manager-gated.

The four phone screens:

1 · PIN lockPick your name from an avatar strip, enter your 4-digit PIN. Start + after every logout.
2 · Table floorZone tabs (Main Bar / Terrace / VIP) + a 2-column grid of table cards.
3 · Table POS2-column product grid with quick-add; a sticky bottom "View tab" bar; the tab itself opens as a bottom sheet.
4 · SettleSingle-column bill grouped by server, VAT, cash / mobile money. Reached via a manager PIN modal for non-managers.

Plus two overlays: a Manager PIN modal (bottom sheet) and a toast. Admin config is intentionally not on mobile — it's a back-office dashboard that lives on the terminal; the lock screen notes "configured by admin on the main terminal."

04 — Device frame & stage

402 × 874, letterboxed

Every screen renders inside <IOSDevice> (default 402×874, radius 48, overflow hidden). The frame draws its own status bar (z 10), dynamic island (z 50) and home indicator (z 60). Screen content sits behind those, so each .m-screen is position:absolute; inset:0 and reserves space with spacers:

  • .m-safe-top — 60px, clears the status bar / island.
  • .m-safe-bottom — 30px, clears the home indicator.

The device is centered and scaled to the viewport by MStage (same guarded scaler as tablet — retries a 0-sized mount, re-runs on load, observes with ResizeObserver), fitting 402+40 × 874+40 so there's a little breathing room:

const s = Math.min(vw / (402 + 40), vh / (874 + 40));
fit.style.transform = "translate(-50%, -50%) scale(" + (s || 1) + ")";
On a real device you'd drop IOSDevice + MStage and let .m-screen fill the viewport, swapping the fixed 60/30 spacers for env(safe-area-inset-*). The internal layout (flex column: fixed header → .m-scroll flex:1 → fixed footer) is already responsive to any height.

06 — Shared data layer

Everything comes from bar/data.jsx

Mobile does not redefine any data. It imports the same exports (see §shared data source and the tablet handover for full shapes):

  • Store: useBarStore() → [store, actions]. Same reducer, same localStorage["flipper-bar-mode-v1"], same fresh-session-on-load rule.
  • Constants: BAR_CASHIERS, BAR_PRODUCTS, and the editable floor in store.floor.
  • Helpers: barCashier, barProduct, findTable(floor,id), barColorFor, barAbbr, barMoney, barFmtClock, barFmtElapsed, tabTotal, tabCount, tabServers.
  • Actions used: login, logout, openTable, addItem, setLineQty, saveToTab, goSettle, elevateSettle, settleConfirm, setScreen. (Mobile does not use the admin-only setSetting / floor-editor actions or setLinePrice.)
Add-item merge & line ownership are unchanged. Each line still carries cashierId; a server edits their own lines, a manager edits any, others are read-only. This drives the grouped tab sheet and grouped settle bill.

07 — State & routing

Same machine, mobile entry

MBarApp maps store.screen to a mobile view. Because admin doesn't exist on mobile, on mount it redirects admin → lock, and treats admin as lock in the switch:

store.screenMobile view
admin / lock / (default)MLock
tablesMFloor
posMPos (tab sheet is local state)
settleMSettle

Transitions match tablet: login → tables; tap table → pos; back / save → tables; logout → lock; settle (manager) → settle; settle (server) → manager modal → elevateSettle → settle; confirm → tables + toast. Guards: no cashier ⇒ lock; pos/settle without a table ⇒ floor.

08 — Interaction spec

Gestures & states
AreaBehaviour
PIN keypadDisabled until a name is tapped; 4th digit auto-submits; wrong shakes + clears. No hardware-keyboard binding on mobile (touch keypad only).
Zone tabsSwitch the visible grid; count badge = open/total per zone.
Add itemTap card or ＋ → increments the current server's default-price line (capped at stock) else adds a new tagged line; in-cart count pill updates; out-of-stock disabled.
Tab sheetOpens on "View tab"; closes on scrim tap, ×, or after Save/Logout/Settle. Grouped by server; own lines editable, others read-only; manager edits all.
Save vs LogoutSave → floor (tab kept). Logout → lock (tab kept, ready for next server). Both dismiss the sheet.
Settle gateServer → manager PIN modal; manager → straight to settle. Disabled on empty tab.
ReloadFresh logged-out session at lock; tabs + settings + floor persist (shared store).
Touch feedbackCards/keys/buttons scale ~0.94–0.98 on :active.

10 — Acceptance checklist

Done means
  • Everything renders inside a 402×874 frame, scaled to fit, with safe-area spacers top & bottom.
  • Lock: avatar strip + keypad; wrong PIN shakes; correct logs in → floor.
  • Floor: zone tabs switch a 2-col grid; open vs free with totals, servers, time.
  • POS: 2-col grid, quick-add, in-cart counts; sticky "View tab" bar shows count + total.
  • Tab sheet: slides up, grouped by server, own lines editable (stepper), others read-only; Save / Logout / Settle work and dismiss the sheet.
  • Server tapping Settle gets the manager PIN modal; manager goes straight through.
  • Settle: grouped bill + VAT, cash/MoMo, confirm closes table + toast → floor.
  • State persists and syncs with the tablet build (same store key); reload lands at lock.
  • All tap targets ≥ 44px; only design-system tokens used.

const { useState: useM, useEffect: useME, useRef: useMR, useMemo: useMM } = React;

/* ---------------- PIN entry (dots + keypad, mobile) ---------------- */
function MPinEntry({ len = 4, onComplete, disabled, errorNonce = 0 }) {
  const [val, setVal] = useM('');
  const [err, setErr] = useM(false);
  const last = useMR(errorNonce);
  useME(() => {
    if (errorNonce !== last.current) {
      last.current = errorNonce; setErr(true);
      const t1 = setTimeout(() => setVal(''), 380);
      const t2 = setTimeout(() => setErr(false), 420);
      return () => { clearTimeout(t1); clearTimeout(t2); };
    }
  }, [errorNonce]);
  const push = (d) => {
    if (disabled) return; setErr(false);
    setVal((v) => { if (v.length >= len) return v; const nv = v + d; if (nv.length === len) setTimeout(() => onComplete(nv), 90); return nv; });
  };
  const back = () => { setErr(false); setVal((v) => v.slice(0, -1)); };
  const clear = () => { setErr(false); setVal(''); };
  return (
    <>
      <div className={`m-dots ${err ? 'err' : ''}`}>
        {Array.from({ length: len }).map((_, i) => <span key={i} className={`m-dot ${i < val.length ? 'filled' : ''}`} />)}
      </div>
      <div className="m-keypad">
        {['1','2','3','4','5','6','7','8','9'].map((k) => <button key={k} className="m-key" disabled={disabled} onClick={() => push(k)}>{k}</button>)}
        <button className="m-key util" disabled={disabled} onClick={clear}>Clear</button>
        <button className="m-key" disabled={disabled} onClick={() => push('0')}>0</button>
        <button className="m-key util" disabled={disabled} onClick={back} aria-label="Delete"><Icons.ChevLeft size={22} /></button>
      </div>
    </>
  );
}

function MStage({ children }) {
  const ref = useMR(null), stageRef = useMR(null);
  useME(() => {
    let raf = 0;
    const fit = () => {
      const el = ref.current; if (!el) return;
      const vw = window.innerWidth, vh = window.innerHeight;
      if (!vw || !vh) { raf = requestAnimationFrame(fit); return; }
      const s = Math.min(vw / (402 + 40), vh / (874 + 40));
      el.style.transform = `translate(-50%, -50%) scale(${s || 1})`;
    };
    fit();
    window.addEventListener('resize', fit); window.addEventListener('load', fit);
    let ro; if (window.ResizeObserver && stageRef.current) { ro = new ResizeObserver(fit); ro.observe(stageRef.current); }
    return () => { cancelAnimationFrame(raf); window.removeEventListener('resize', fit); window.removeEventListener('load', fit); if (ro) ro.disconnect(); };
  }, []);
  return <div className="mbar-stage" ref={stageRef}><div className="mbar-fit" ref={ref}>{children}</div></div>;
}

function useBarStore() {

        __lid = max + 1;
        // back-fill floor for state saved before the floor editor existed
        if (!parsed.floor) parsed.floor = cloneFloor(BAR_ZONES);
        // persist tabs + settings + floor, but always start a fresh session
        // logged-out at the admin entry so a reload never lands mid-flow.
        return { ...parsed, screen: 'admin', activeCashierId: null, activeTableId: null };
      }
    } catch (e) {}
    return seedState();
  });

  React.useEffect(() => {
    try { localStorage.setItem(BAR_STORE_KEY, JSON.stringify(state)); } catch (e) {}
  }, [state]);

  const patch = (p) => setState((s) => ({ ...s, ...(typeof p === 'function' ? p(s) : p) }));

    })),

    openTable: (tableId) => setState((s) => {
      const tabs = { ...s.tabs };

      }
      return { ...s, tabs, activeTableId: tableId, screen: 'pos' };
    }),

      const idx = lines.findIndex((l) => l.productId === product.id && l.cashierId === s.activeCashierId && l.price === product.price);
      if (idx >= 0) lines[idx] = { ...lines[idx], qty: Math.min(product.stock, lines[idx].qty + 1) };
      else lines.push({ id: lid(), productId: product.id, qty: 1, price: product.price, cashierId: s.activeCashierId, ts: Date.now() });

      }).filter((l) => l.qty > 0);

    setLinePrice: (lineId, price) => setState((s) => {

    addTable: (zoneId) => setState((s) => {

        if (z.id !== zoneId) return z;
        const prefix = ((z.tables[0] && z.tables[0].id.match(/^[A-Za-z]+/)) || [null])[0]
          || (z.name.replace(/[^A-Za-z]/g, '')[0] || 'T').toUpperCase();
        let n = z.tables.length + 1, id;
        do { id = prefix + n; n++; } while (ids.has(id));
        ids.add(id);
        return { ...z, tables: [...z.tables, { id, seats: 4 }] };
      });
      return { ...s, floor };
    }),
    setTableSeats: (tableId, seats) => setState((s) => ({

        ...z, tables: z.tables.map((t) => t.id === tableId ? { ...t, seats: Math.max(1, Math.min(30, seats)) } : t),
      })),
    })),
    removeTable: (tableId) => setState((s) => ({
      ...s, floor: s.floor.map((z) => ({ ...z, tables: z.tables.filter((t) => t.id !== tableId) })),
    })),
  }), []);

  return [state, actions];
}
