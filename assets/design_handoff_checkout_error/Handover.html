“Flo” — Developer Handover

Everything needed to implement the redesigned chat panel: design tokens, every component, the answer-block contract, the WhatsApp connect + inbox flows, backend integration points, and the complete annotated source (CSS + JSX) inlined at the bottom. Goal: match SAP Joule’s capability in Flipper’s brand. Scope is the chat panel only — no global left nav.

1 Overview & goals

One panel hosts two modes toggled in the header:

  • Ask Flo — a conversational AI that answers business questions from live data with rich cards (metrics, charts, tables), source grounding, follow-ups, and a proactive daily briefing.
  • Messages — a WhatsApp inbox where customer chats land and Flo drafts replies grounded in your data.

Currency is RWF; locale is Rwanda (EBM tax context). Data source is MiniData (a Supabase project: sales, users, products, payments). Reference product: SAP Joule.

Naming

The assistant is Flo (“Flipper flow”). Brand mark = the Flipper swirl on a cyan→blue→indigo gradient. The mark doubles as the bot avatar.

Animation gotcha (learned the hard way)

Never rely on an entrance animation that starts at opacity:0/scaleY(0) for content that must be visible — some capture/print contexts freeze the animation timeline at frame 0 and the content vanishes. Make the rest state visible; treat animations as optional enhancement (gate behind prefers-reduced-motion: no-preference). Charts render at full state; modals appear without fade.

4 Panel shell & surfaces

The chat is a single flex column: header (fixed) → body → composer/inbox. Modals overlay the panel (position:absolute; inset:0) — not the viewport.

  • Desktop: centered card ~1040×860, radius 24, --sh-3.
  • Mobile: 412×892 device frame (status bar + notch). Add class is-mobile on the chat root; it drives all mobile rules (hide identity text + chips, stack inbox, 2-col metrics, etc.).

In the prototype, surface is a Tweak (Desktop/Mobile). In production, make it responsive: apply is-mobile below ~640px.

5 Header & modes

Left→right:

  1. Identity: brand mark + “Flo” + Business AI + status ● MiniData connected · live (green live dot). Hidden on mobile.
  2. Mode toggle (segmented, centered): ✦ Ask Flo ⌾ Messages. Messages shows a green unread badge — only when WhatsApp is connected and unread>0.
  3. Channel chips (hidden on mobile): ⛁ MiniData (green dot) and either a dashed Connect WhatsApp or solid ⌾ WhatsApp when connected. Both open their modal.
  4. New chat button (hidden on mobile — redundant with menu).
  5. “+” menu popover (closes on outside click): New conversation · Connect WhatsApp (On/Off badge) · Manage data sources.

6 Ask Flo — home / empty state

Shown when the Ask-Flo thread is empty:

  • Greeting: eyebrow (day · time), headline “Good evening, {Shop}. Ask me anything…” (the em word uses --grad-brand as clipped text), sub paragraph.
  • Daily briefing card (proactive, generated daily): badge + auto timestamp, headline, paragraph with bold figures, 3 mini-stats (Revenue / Net profit / Units, each with up/down delta).
  • Try asking grid (2-col desktop, 1-col mobile) of suggestion cards {icon, tone, title, desc, q} → clicking sends q.
  • Channels grid: MiniData (Connected · Manage→sources) + WhatsApp (Connected/Not set up · Connect/Manage).

7 Conversation & thinking

  • User row: right-aligned bubble, --grad-btn, radius 20/20/6/20, white text.
  • Bot row: brand-mark avatar + a column that renders the answer-block array (gap 13).
  • Thinking: before the answer, show the avatar + a vertical step list animating done→active→pending (tick / spinner / dashed). ~0.4–0.5s/step; in production stream these from your real query/tool phases (e.g. “Querying MiniData · sales”).
  • Auto-scroll to bottom on each new message/block.

8 Answer-block schema core contract

An answer is an ordered array of typed blocks. Your LLM/back end returns this JSON; the renderer maps each type to a component. Implement all eight. Always include a source block (grounding) and, when relevant, followups.

typeshapenotes
text{html}Allows <b>, <span class="pos|neg">.
metrics{cols:2|3|4, items:[{k,v,unit?,delta?,dir:'up'|'down'|'flat',neg?,hl?}]}Tiles; value mono; dir→arrow+color; hl=blue tint; mobile forces 2-col.
viz (bar){kind:'bar',title,sub,data:[{label,value,tone:'blue'|'gain'|'loss',display}]}Signed bars + zero baseline; negative value label sits inside the bar.
viz (area){kind:'area',title,sub,points:number[],labels:string[],peak:index}7-day line+area, peak marker.
table{cols:[{label,num?}],rows:[cell...]}cell = string | {t,cls:'pos'|'neg'} | {lead,sw:'#hex'}. num cols right-align mono.
callout{tone:'warn'|'info',h,p}Icon chip + heading + body.
source{items:[...]}Grounding chip; first item bold (e.g. MiniData · sales · today).
actions{items:[{label,icon,primary?}]}e.g. View EOD report, Export.
followups{items:[{label,q}]}Chips → send q.

10 WhatsApp connect flow

Four entry points all open the connect modal: header chip, “+” menu, home Channels card, and the Messages locked-state button.

Connect modal (not connected)

WA-green header; intro; 3 numbered steps (Open Meta Business Suite ↗ · Find Phone Number ID 🔍 · Paste & connect ⧉); Phone Number ID input (mono/numeric); privacy hint; Connect WhatsApp (disabled until ≥6 chars).

Success state (connected)

Green check, “You’re connected”, account row (number + “Phone Number ID · {shop}” + ● Live), Disconnect button.

On connect, persist {connected:true, phoneNumberId}; the header chip turns solid green, the menu badge flips On, and the Messages tab becomes usable.

11 Messages inbox

Not connected → a locked state: WA icon, “Answer customers on WhatsApp”, explainer, Connect button, 3-step strip. This is the answer to “how do I see WhatsApp messages?” — they land here once connected.

Connected → master-detail (both panes on desktop; stacked with a back arrow on mobile):

  • List (~326px): header “Customers / N unread / via WhatsApp”; rows = avatar(initials+tone) + name + time(mono) + last-message snippet (prefix “You: ” for outbound) + green unread badge. Selecting clears that thread’s unread.
  • Thread: head (back on mobile + avatar + name + WhatsApp · {phone} + actions); messages — inbound = neutral left bubble, outbound = WA-green right bubble; “Today” divider; mono times.
  • Flo suggested reply card (when last message is inbound and a draft exists): mark + “Flo suggested reply” + source chip + draft text + Send reply / Edit first / Dismiss.
  • Reply composer: optional Draft button (loads Flo’s draft), textarea (Enter=send), green send.

Inbox item shape: {id, name, initials, tone, phone, time, unread, last, msgs:[{from:'them'|'me',t,time}], flo:{src,draft}|null}. See chat-inbox.jsx.

12 Backend contracts (build for real)

MiniData = Supabase

Tables: sales, users, products, payments. The Data Sources modal lists/adds/edits/removes connections (status Active). Flo’s answers, the daily briefing, and reply drafts all query this.

Flo answers (Ask Flo)

Pipeline: user question → retrieve schema + run read-only queries over MiniData (RAG / tool-calls) → LLM composes the block array (§8) as JSON. Stream the thinking steps from the query phases. Rules: format money as RWF, mono numerals; always attach a source block; never fabricate — when data is missing, return a callout saying so (mirrors today’s “No tax/traffic data” case).

WhatsApp (Meta Cloud API)

  • Connect: store Phone Number ID + access token. Prefer Meta Embedded Signup/OAuth; the manual “Phone Number ID” paste matches the current prototype.
  • Inbound: webhook → upsert message into the thread keyed by wa_id/phone → realtime push to the inbox. Unread = inbound since last read.
  • Outbound: POST to the messages endpoint; append as {from:'me'}.
  • Drafts: on inbound, call the LLM with the customer message + MiniData context → {src, draft}. Roadmap: live regenerate, auto-reply toggle for FAQs (hours/stock), canned quick-replies, customer order side-panel.

13 State model & acceptance

Session state: mode:'flo'|'inbox' · messages[] (Ask-Flo thread) · wa:{connected,phoneNumberId} · modal:null|'whatsapp'|'sources' · menuOpen · inbox threads{id:{msgs,unread}}, selected + per-thread reply input & floState:'open'|'used'|'dismissed'.

Acceptance checklist

  • Empty Ask-Flo shows Home; sending hides Home, reveals thread + quick prompts; thinking always resolves to a visible answer.
  • Charts/areas render at rest (no opacity-0 dependency); modals appear without relying on animation.
  • Messages badge only when connected + unread; disconnected shows locked state.
  • Mobile: list↔thread with back; header collapses; metrics 2-col; composer clears the home indicator.
  • Connect needs ID ≥6 chars; connection reflects in header chip, menu, home card, Messages.
  • Negative profit = red + minus; all money RWF + mono.
Mocked in the prototype → build real

Scripted Flo answers & drafts (→ live LLM + Supabase); simulated WhatsApp connect (→ Cloud API + webhooks); static inbox seed (→ live threads); no auth/persistence.

// ---- Triple bar: Revenue / Profit (can be negative) / Units ----
function BarChart({ data, unit = 'RWF' }) {
  // data: [{label, value, tone}]  tone: 'blue'|'gain'|'loss'
  const W = 520, H = 220, padX = 16, padTop = 24, padBot = 38;
  const plotH = H - padTop - padBot;
  const max = Math.max(...data.map(d => d.value), 0);
  const min = Math.min(...data.map(d => d.value), 0);
  const span = (max - min) || 1;
  const zeroY = padTop + (max / span) * plotH;
  const n = data.length;
  const slot = (W - padX * 2) / n;
  const bw = Math.min(78, slot * 0.42);
  const colorOf = (t) => t === 'loss' ? '#E5484D' : t === 'gain' ? '#10B981' : '#2563EB';
  const fmt = (v) => {
    const a = Math.abs(v);
    if (a >= 1000) return (v / 1000).toFixed(a >= 10000 ? 0 : 1) + 'K';
    return String(v);
  };
  // gridlines (include a solid zero baseline when bars span 0)
  const mid = (min < 0 && max > 0) ? 0 : (max + min) / 2;
  const lines = [max, mid, min].filter((v, i, arr) => arr.indexOf(v) === i);
  return (
    <svg viewBox={`0 0 ${W} ${H}`} role="img">
      {/* gridlines */}
      {lines.map((lv, i) => {
        const y = padTop + ((max - lv) / span) * plotH;
        return (
          <g key={i}>
            <line x1={padX} x2={W - padX} y1={y} y2={y} stroke="#EDF1F7" strokeWidth="1" strokeDasharray={lv === 0 ? '0' : '3 4'} />
            <text x={padX} y={y - 4} fontSize="10" fill="#AEB8CA" fontFamily="'Geist Mono',monospace">{fmt(lv)}</text>
          </g>
        );
      })}
      {data.map((d, i) => {
        const cx = padX + slot * i + slot / 2;
        const v = d.value;
        const topY = padTop + ((max - Math.max(v, 0)) / span) * plotH;
        const botY = padTop + ((max - Math.min(v, 0)) / span) * plotH;
        const h = Math.max(2, botY - topY);
        const c = colorOf(d.tone);
        const isNeg = v < 0;
        const labelY = isNeg ? botY - 10 : topY - 7;
        return (
          <g key={i}>
            <rect x={cx - bw / 2} y={topY} width={bw} height={h} rx="6" fill={c} />
            <text x={cx} y={labelY} fontSize="11.5" fontWeight="700" textAnchor="middle"
              fill={isNeg ? '#fff' : c} fontFamily="'Geist Mono',monospace">{d.display ?? fmt(v)}</text>
            <text x={cx} y={H - 16} fontSize="11.5" textAnchor="middle" fill="#7E8AA0" fontWeight="600" fontFamily="'Geist',sans-serif">{d.label}</text>
          </g>
        );
      })}
    </svg>
  );
}

function AreaChart({ points, labels, peak }) {
  const W = 520, H = 200, padL = 8, padR = 8, padTop = 18, padBot = 30;
  const plotW = W - padL - padR, plotH = H - padTop - padBot;
  const max = Math.max(...points) * 1.12, min = Math.min(...points) * 0.78;
  const span = (max - min) || 1;
  const xs = points.map((_, i) => padL + (i / (points.length - 1)) * plotW);
  const ys = points.map(v => padTop + ((max - v) / span) * plotH);
  const line = xs.map((x, i) => `${i === 0 ? 'M' : 'L'}${x.toFixed(1)} ${ys[i].toFixed(1)}`).join(' ');
  const area = `${line} L ${xs[xs.length - 1].toFixed(1)} ${padTop + plotH} L ${xs[0].toFixed(1)} ${padTop + plotH} Z`;
  const totLen = 900;
  return (
    <svg viewBox={`0 0 ${W} ${H}`} role="img">
      <defs>
        <linearGradient id="areaFill" x1="0" y1="0" x2="0" y2="1">
          <stop offset="0" stopColor="#2563EB" stopOpacity="0.20" />
          <stop offset="1" stopColor="#2563EB" stopOpacity="0" />
        </linearGradient>
      </defs>
      {/* gridlines */}
      {[0, 0.5, 1].map((t, i) => {
        const y = padTop + t * plotH;
        return <line key={i} x1={padL} x2={W - padR} y1={y} y2={y} stroke="#EDF1F7" strokeWidth="1" strokeDasharray="3 4" />;
      })}
      <path d={area} fill="url(#areaFill)" />
      <path d={line} fill="none" stroke="#2563EB" strokeWidth="2.5" strokeLinecap="round" strokeLinejoin="round" />
      {xs.map((x, i) => {
        const isPeak = i === peak;
        return (
          <g key={i}>
            <circle cx={x} cy={ys[i]} r={isPeak ? 5 : 3.5} fill="#fff" stroke="#2563EB" strokeWidth={isPeak ? 3 : 2} />
            {isPeak && <text x={x} y={ys[i] - 12} fontSize="11" fontWeight="700" textAnchor="middle" fill="#1D4ED8" fontFamily="'Geist Mono',monospace">peak</text>}
            <text x={x} y={H - 10} fontSize="11" textAnchor="middle" fill="#7E8AA0" fontWeight="600" fontFamily="'Geist',sans-serif">{labels[i]}</text>
          </g>
        );
      })}
    </svg>
  );
}

let MID = 0;
function Chat({ isMobile }) {
  const [msgs, setMsgs] = useState([]);
  const [mode, setMode] = useState('flo'); // 'flo' | 'inbox'
  const [modal, setModal] = useState(null);
  const [menuOpen, setMenuOpen] = useState(false);
  const [wa, setWa] = useState({ connected: false, number: '' });
  const scRef = useRef(null);
  const pendingRef = useRef({});
  const newChat = () => { setMsgs([]); pendingRef.current = {}; };

  const scrollDown = useCallback(() => {
    const el = scRef.current; if (!el) return;
    requestAnimationFrame(() => { el.scrollTop = el.scrollHeight; });
  }, []);

  useEffect(() => { scrollDown(); }, [msgs, scrollDown]);

  const ask = useCallback((q, customText) => {
    const answer = (q && ANSWERS[q]) || FALLBACK;
    const userText = customText || PROMPT_TEXT[q] || q || '';
    const botId = ++MID;
    pendingRef.current[botId] = answer;
    setMsgs(m => [
      ...m,
      { id: ++MID, role: 'user', text: userText },
      { id: botId, role: 'bot', state: 'thinking', think: answer.think },
    ]);
  }, []);

  const finishBot = useCallback((botId) => {
    const answer = pendingRef.current[botId];
    if (!answer) return;
    setMsgs(m => m.map(x => x.id === botId ? { ...x, state: 'done', blocks: answer.blocks } : x));
    delete pendingRef.current[botId];
  }, []);

  const empty = msgs.length === 0;
  const unread = wa.connected ? INBOX.reduce((s, c) => s + c.unread, 0) : 0;

  return (
    <div className={'chat' + (isMobile ? ' is-mobile' : '')}>
      <div className="chat-head">
        <div className="flo-mark"><FloMark size={22} /></div>
        <div className="chat-head-id">

        </div>
        <div className="chat-head-spacer" />
        <div className="mode-tabs">

          <button className={'mode-tab' + (mode === 'inbox' ? ' is-on' : '')} onClick={() => setMode('inbox')}><Ic.WhatsApp size={15} />Messages{unread > 0 && <span className="tab-badge">{unread}</span>}</button>
        </div>
        <div className="chat-head-spacer" />
        <div className="chan-chips">
          <span className="chan-chip minidata on"><span className="cc-dot" /><Ic.Database size={14} />MiniData</span>
          {wa.connected
            ? <span className="chan-chip wa on" onClick={() => setModal('whatsapp')} role="button"><span className="cc-dot" /><Ic.WhatsApp size={14} />WhatsApp</span>
            : <button className="chan-chip connect" onClick={() => setModal('whatsapp')}><Ic.WhatsApp size={14} />Connect WhatsApp</button>}
        </div>
        <button className="head-btn newchat-btn" onClick={newChat}><Ic.NewChat size={17} />{!isMobile && 'New chat'}</button>
        <div className="head-menu-wrap">
          <button className="head-btn icon" onClick={() => setMenuOpen(o => !o)} title="Menu"><Ic.Plus size={17} /></button>
          {menuOpen && (
            <>
              <div style={{ position: 'fixed', inset: 0, zIndex: 55 }} onClick={() => setMenuOpen(false)} />
              <div className="menu-pop">
                <button className="menu-item" onClick={() => { newChat(); setMenuOpen(false); }}>
                  <span className="mi-ico" style={{ background: 'var(--blue-tint)', color: 'var(--blue)' }}><Ic.NewChat size={16} /></span>
                  <span className="mi-txt">New conversation</span>
                </button>
                <div className="menu-sep" />
                <button className="menu-item" onClick={() => { setModal('whatsapp'); setMenuOpen(false); }}>
                  <span className="mi-ico" style={{ background: 'var(--wa-tint)', color: 'var(--wa-deep)' }}><Ic.WhatsApp size={16} /></span>
                  <span className="mi-txt">Connect WhatsApp<span className="mi-sub">Chat with Flo &amp; customers</span></span>
                  <span className={'menu-badge ' + (wa.connected ? 'on' : 'off')}>{wa.connected ? 'On' : 'Off'}</span>
                </button>
                <button className="menu-item" onClick={() => { setModal('sources'); setMenuOpen(false); }}>
                  <span className="mi-ico" style={{ background: 'var(--blue-tint)', color: 'var(--blue)' }}><Ic.Database size={16} /></span>
                  <span className="mi-txt">Manage data sources<span className="mi-sub">MiniData · Supabase</span></span>
                </button>
              </div>
            </>
          )}
        </div>
      </div>

      {mode === 'flo' ? (
        <>
          <div className="chat-scroll" ref={scRef}>
            <div className="chat-thread">
              {empty && <Home ask={ask} isMobile={isMobile} channels={{ whatsapp: wa.connected }} openModal={setModal} />}
              {msgs.map(m => (
                m.role === 'user'
                  ? <div key={m.id} className="msg msg-user"><div className="bubble">{m.text}</div></div>
                  : <BotMessage key={m.id} msg={m} ask={ask} isMobile={isMobile} onDone={() => finishBot(m.id)} />
              ))}
            </div>
          </div>

          <Composer onSend={ask} showQuick={!empty} isMobile={isMobile} />
        </>
      ) : (
        <Inbox isMobile={isMobile} connected={wa.connected} onConnect={() => setModal('whatsapp')} />
      )}

      {modal === 'whatsapp' && (
        <Modal onClose={() => setModal(null)}>
          <WhatsAppConnect connected={wa.connected} number={wa.number}
            onConnect={(n) => setWa({ connected: true, number: n })}
            onClose={() => setModal(null)}
            onDisconnect={() => { setWa({ connected: false, number: '' }); setModal(null); }} />
        </Modal>
      )}
      {modal === 'sources' && (
        <Modal onClose={() => setModal(null)}><DataSources onClose={() => setModal(null)} /></Modal>
      )}
    </div>
  );
}
