Complete build spec for the subscription plan-selection screen: token stylesheet, per-component markup & rules, states, business logic, the one layout gotcha, and a Flutter widget mapping. 01 — Overview

What this screen does

A single-column plan-selection screen inside a phone frame with a fixed header. The user picks a billing cycle, selects one of three plans, optionally adds a service, optionally splits the charge into installments, and confirms. It reuses the Flipper checkout component set (segmented control, plan tiles, toggle switch, primary button) so it stays visually consistent with the rest of the app.

  • Primary goal — choose a plan and proceed to payment.
  • Live pricing — the total recomputes on every cycle / plan / add-on / installment change.
  • Payment rail — MTN Mobile Money (shown in the CTA sub-note; the charge itself happens on the next screen).
Files. New styles live in payment_plan/plan.css; shared component styles in payment_issue/payment.css; base tokens in onboarding/styles.css. Behaviour reference is payment_plan/plan-app.jsx — this handover restates the same UI as framework-free HTML/CSS plus a Flutter map.

17 — Scroll & layout gotcha

Read this before you build

The screen is a flex column: a fixed .pi-top, then a .pi-scroll region with flex:1; min-height:0; overflow-y:auto. Blocks inside are separated by gap:16px.

Web gotcha. Because .pi-scroll is a flex column, its children can shrink below content height and any card with overflow:hidden will clip to a sliver. Fix: .pi-scroll > * { flex-shrink:0; }. In Flutter this doesn't apply — a ListView/SingleChildScrollView already gives children their natural height.
Flutter equivalent. Column (fixed header) → Expanded(child: ListView(padding: EdgeInsets.fromLTRB(20,4,20,26), children: [...])). Insert SizedBox(height:16) between blocks (or separatorBuilder).

Number formatting

Use intl: NumberFormat.decimalPattern('en_US').format(value) for grouped thousands, then append .00 for the total. Keep every numeric string in the GeistMono font.

final nf = NumberFormat.decimalPattern('en_US');
final total = computeTotal(plan, yearly, addonOn);
final label = 'RWF ${nf.format(total.round())}.00';   // RWF 1,152,000.00
Animation curves. Segment/switch thumbs use cubic-bezier(.22,.9,.3,1) ≈ Curves.easeOutCubic at 220–260ms. The selected-plan check "pops" with cubic-bezier(.2,1.5,.4,1) ≈ an overshoot (Curves.easeOutBack) at ~300ms. Gate any entrance animation so the resting state is always visible (matters for reduced-motion & instant loads).
